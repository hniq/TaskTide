# DDL 任务规划器 - 完整规格文档

## 概述

"不咕了"是一个面向拖延症人群的 Web 任务管理工具，采用深色极简设计风格，基于艾森豪威尔矩阵进行任务优先级管理。核心功能包括：智能 AI 任务解析、自动任务拆分、DDL 优先级排序、每日任务自动分配。

## 技术架构

### 前端
- **技术栈**: 原生 HTML5 + CSS3 + ES6+ JavaScript (模块化)
- **UI 风格**: 深色极简 (参考 Linear/Notion 暗色模式)
- **存储**: localStorage 本地持久化

### 后端
- **技术栈**: Node.js + Express
- **AI 服务**: 魔搭 (ModelScope) API - 通义千问模型
- **部署**: 魔搭创空间 (监听 0.0.0.0:7860)

### 安全
- API Key 通过环境变量 `DASHSCOPE_KEY` 注入后端
- 前端绝不接触 API Key，仅通过后端代理调用

## 数据模型

### Task 对象
```js
{
  id: "唯一ID",
  title: "任务标题",
  deadline: "2026-02-20",       // 可为 null（无截止日期）
  urgency: 3,                   // 紧急程度 1-5
  importance: 3,                // 重要程度 1-5
  estimatedHours: 8,            // 预估工时
  eisenhowerQuadrant: "Q1",     // 自动计算: Q1|Q2|Q3|Q4
  status: "active",             // active | completed
  subtasks: [{
    id: "子任务ID",
    title: "子任务名",
    estimatedHours: 2,
    completed: false,
    order: 0,
    _manuallyAssigned: false,   // 是否被手动拖拽分配到日期
    _assignedDate: null         // 手动分配的日期
  }],
  notes: "",
  manualPriority: null,
  manualOrder: 0,               // 手动排序序号
  assignedDays: {},             // 日期 -> { subtaskIds: [], hours: N } 或 数字(旧格式)
  createdBy: "manual",          // manual | ai
  completedAt: null,            // 完成时间 ISO 字符串
  aiContext: null
}
```

### Settings 对象
```js
{
  dailyWorkHours: 4,
  splitThreshold: 6,
  autoAssignAfterSplit: true,
  defaultCalendarView: 'month',
  defaultSortMode: 'deadline',  // deadline | priority | manual
  habitMemory: { taskPatterns: [] }
}
```

## 核心算法

### 1. 艾森豪威尔矩阵分类
```
urgency >= 3.5 && importance >= 3.5 → Q1 (紧急+重要，红色)
urgency < 3.5 && importance >= 3.5 → Q2 (重要不紧急，蓝色)
urgency >= 3.5 && importance < 3.5 → Q3 (紧急不重要，橙色)
其他 → Q4 (不紧急不重要，灰色)
```

### 2. 优先级评分
```
基础分: Q1=95, Q2=75, Q3=55, Q4=35
加分: DDL临近加分 max(0, (7-daysLeft)*2)
已逾期: 额外+20分
```

### 3. 每日分配算法 (贪心)
- 按优先级从高到低处理任务
- 收集已手动分配的子任务ID集合（通过 _manuallyAssigned 标记和 assignedDays 中已有的 subtaskIds）
- 仅对未手动分配且未完成的子任务执行自动分配
- 从今天到 DDL 前一天，依次填充每天的时间槽
- 每天不超过用户设定的可用时长
- 超载/无法完成时显示警告

### 3.1 无截止日期任务分配策略
- 对于无截止日期的任务，根据四象限智能分配：
  - Q2（重要不紧急）/ Q4（不重要不紧急）→ 分配到本周末（周六/周日）
  - Q1（紧急重要）/ Q3（紧急不重要）→ 分配到未来两周内
- 单任务自动分配：详情页"自动分配"按钮可为单个任务执行分配，同样遵循上述策略

### 4. 排序算法
- 按截止日期：无截止日期排最后，其余按日期升序
- 按优先级：使用 calcPriority 动态分数降序
- 手动排序：按 manualOrder 字段升序，支持拖动调整

## AI 功能

### 工作流程
1. 用户在 AI 输入框输入自然语言描述
2. 后端调用魔搭 API 解析任务信息
3. **前端展示预览卡片**，用户可编辑所有字段（标题、截止日期、紧急/重要程度、预估时长、子任务）
4. 用户点击"确认创建"后才实际创建任务
5. 如果设置了"拆分后自动分配"，创建后自动执行分配

### API 端点
- `POST /api/ai/parse-task` - 解析自然语言任务描述
- `GET /api/ai/health` - 服务健康检查

### 请求格式
```json
{
  "text": "完成课程论文，3天后截止，大概需要8小时"
}
```

### 响应格式
```json
{
  "title": "完成课程论文",
  "deadline": "2026-02-25",
  "urgency": 4,
  "importance": 4,
  "estimatedHours": 8,
  "subtasks": [
    { "title": "资料收集", "estimatedHours": 2 },
    { "title": "撰写初稿", "estimatedHours": 4 },
    { "title": "修改完善", "estimatedHours": 2 }
  ],
  "reasoning": "课程论文属于重要学习任务，3天后截止较紧急"
}
```
